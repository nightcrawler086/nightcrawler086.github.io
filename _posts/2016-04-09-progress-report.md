## Progress Report - March 2016

For the last month, I've been entirely focused on AWS (Amazon Web Services), and it's been interesting to say the least.  Coming from a traditional infrastructure background, it felt like going into enemy territory.  The vendors and products in which I specialized all but declared AWS (and public clouds like them) public enemy number one.  Now that I'm learning more about them, I definitely understand why.

I finally gave in and purchased a subscription to [Linux Academy](linuxacademy.com).  Their content on AWS alone has been worth it.  I'm excited for a lot of the other content there as well. 

I also feel like I'm maturing.  I used to really drink to kool-aid of the vendors that produced the products I worked with, which included all the [FUD](https://en.wikipedia.org/wiki/Fear,_uncertainty_and_doubt).  I can remember debating different solutions over some low-level technical feature or detail.  I'm becoming focused on the entire solution, and how well that solution solves the problem.  It's also taught me not to ignore any technology.  You need to understand the technology before trying to discredit it. 

I've noticed that I end up wasting a lot of time when I sit down and don't have a very long time to train.  I usually like to have at least an hour, but it's not uncommon for me to only get 30 minutes or so.  I wanted to find something to fill those little chunks of time, so  I'm going to use that time to pick up a programming language.  I don't think you learn a programming language the same way you could learn a specific product.  You can learn in little pieces over a long period of time.  I haven't completely decided which language I want to learn yet, but it will most likely be [Python](www.python.org).  I also want to learn [Go](golang.org) at some point.
